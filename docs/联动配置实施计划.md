## 一、总体策略

### 分三步走

```
第一步（建骨架）：数据库表 + 后端 CRUD → 数据能存能取
```

### 2.2 新建 `backend/routes/linkage.py`

遵循现有路由文件的结构模式（参考 [routes/finance.py](file:///Volumes/Elements/code/houqinBP/backend/routes/finance.py)）：

| 参考项 | 做法 |
|--------|------|
| 导入 | `from database import get_db, transaction` |
| 认证 | `from auth import require_permission` |
| 模型 | Pydantic `BaseModel` 定义请求体 |
| CRUD | 手写 SQL，无 ORM |
| 并发 | 乐观锁 `version` 字段 |
| 日志 | `log_operation()` 记录操作 |

需要实现 7 个端点：

#### 端点 1：`GET /api/linkage/mappings`

```
功能：返回全部联动配置，附带专业项目名称和 IT 项目名称
SQL：
  SELECT lm.*,
         pp.name AS proj_name
  FROM linkage_mappings lm
  LEFT JOIN professional_projects pp ON pp.id = lm.proj_id
  ORDER BY lm.id

返回后处理：
  1. 将 it_ids 从 JSON 字符串还原为数组
  2. 根据 it_ids 查询 it_projects 获取 it_names 数组
  3. 将 fin_subjects 从 JSON 字符串还原为数组
```

#### 端点 2：`GET /api/linkage/mappings/{id}`

```
功能：获取单条联动配置详情
SQL：同上 + WHERE lm.id = ?
```

#### 端点 3：`POST /api/linkage/mappings`

```
请求体：{ proj_id, it_ids, hr_change_desc, hr_headcount, hr_posts,
          hr_month_start, hr_month_end, fin_budget_impact,
          fin_subjects, fin_description, responsible_person }

后端处理：
  1. 校验 proj_id 在 professional_projects 中存在
  2. 校验 it_ids 中的每个 ID 在 it_projects 中存在
  3. it_ids 和 fin_subjects 序列化为 JSON 字符串存入
  4. 自增 ID
  5. 记录操作日志
```

#### 端点 4：`PUT /api/linkage/mappings/{id}`

```
与 POST 相同的校验逻辑 + 乐观锁版本校验
```

#### 端点 5：`DELETE /api/linkage/mappings/{id}`

```
直接删除，记录操作日志
```

#### 端点 6：`GET /api/linkage/summary`

```
功能：聚合计算所有联动配置的总体影响
SQL（伪代码）：
  SELECT
    COUNT(*) AS total_mappings,
    SUM(hr_headcount) AS total_hr_impact,
    SUM(fin_budget_impact) AS total_budget_impact,
    SUM(CASE WHEN hr_headcount > 0 THEN hr_headcount ELSE 0 END) AS hr_increase,
    SUM(CASE WHEN hr_headcount < 0 THEN hr_headcount ELSE 0 END) AS hr_decrease,
    SUM(CASE WHEN fin_budget_impact > 0 THEN fin_budget_impact ELSE 0 END) AS budget_increase,
    SUM(CASE WHEN fin_budget_impact < 0 THEN fin_budget_impact ELSE 0 END) AS budget_decrease
  FROM linkage_mappings

然后统计：
  - 专业项目覆盖率：COUNT(DISTINCT proj_id) vs 总数 17
  - IT 项目关联数：遍历 it_ids JSON 去重计数
  - 受影响科目：遍历 fin_subjects JSON 聚合求和
```

#### 端点 7：`POST /api/linkage/impact-simulate`

```
功能：模拟某个项目增减 N 人对财务预算的影响

请求体：{ proj_id: int, hr_change: int }

计算逻辑（部门规定人均综合成本 8500 元/月）：
  1. 查询项目名称
  2. 人均成本 = 10.2（万元/年，即 8500 元/月）
  3. 总影响 = hr_change × 10.2
  4. 返回简化结果：{ proj_id, proj_name, hr_change, per_person_cost: 10.2, total_impact }
```

## 四、第三步：出价值（计算与集成）

### 4.1 联动摘要 `GET /api/linkage/summary` 的后端实现细节

```python
@router.get("/summary")
def get_summary(current_user: dict = Depends(require_permission("linkage:read"))):
    db = get_db()
    cur = db.cursor()

    # 聚合统计
    agg = dict(cur.execute("""
        SELECT
            COUNT(*) AS total_mappings,
            COALESCE(SUM(hr_headcount), 0) AS total_hr_impact,
            COALESCE(SUM(fin_budget_impact), 0) AS total_budget_impact,
            COALESCE(SUM(CASE WHEN hr_headcount > 0 THEN hr_headcount ELSE 0 END), 0) AS hr_inc,
            COALESCE(SUM(CASE WHEN hr_headcount < 0 THEN hr_headcount ELSE 0 END), 0) AS hr_dec,
            COALESCE(SUM(CASE WHEN fin_budget_impact > 0 THEN fin_budget_impact ELSE 0 END), 0) AS budget_inc,
            COALESCE(SUM(CASE WHEN fin_budget_impact < 0 THEN fin_budget_impact ELSE 0 END), 0) AS budget_dec
        FROM linkage_mappings
    """).fetchone())

    # 专业项目覆盖率
    total_proj = cur.execute("SELECT COUNT(*) FROM professional_projects").fetchone()[0]
    mapped_proj = cur.execute(
        "SELECT COUNT(DISTINCT proj_id) FROM linkage_mappings WHERE proj_id IS NOT NULL"
    ).fetchone()[0]

    # IT 项目关联数 — 遍历所有 it_ids JSON 数组去重
    rows = cur.execute("SELECT it_ids FROM linkage_mappings WHERE it_ids IS NOT NULL AND it_ids != '[]'").fetchall()
    it_set = set()
    import json
    for (it_ids_str,) in rows:
        try:
            for iid in json.loads(it_ids_str):
                it_set.add(iid)
        except Exception:
            pass

    # 受影响科目聚合
    sub_rows = cur.execute(
        "SELECT fin_subjects, fin_budget_impact FROM linkage_mappings "
        "WHERE fin_subjects IS NOT NULL AND fin_subjects != '[]'"
    ).fetchall()
    subject_map = {}
    for (subs_str, impact) in sub_rows:
        try:
            for sub in json.loads(subs_str):
                subject_map[sub] = subject_map.get(sub, 0) + (impact or 0)
        except Exception:
            pass

    db.close()

    return {
        "total_mappings": agg["total_mappings"],
        "total_hr_impact": agg["total_hr_impact"],
        "total_budget_impact": round(agg["total_budget_impact"], 2),
        "by_dimension": {
            "hr_impact": {
                "increase": agg["hr_inc"],
                "decrease": agg["hr_dec"],
                "net": agg["total_hr_impact"],
            },
            "budget_impact": {
                "increase": agg["budget_inc"],
                "decrease": agg["budget_dec"],
                "net": round(agg["total_budget_impact"], 2),
            },
        },
        "affected_subjects": [
            {"subject": k, "amount": round(v, 2)}
            for k, v in sorted(subject_map.items(), key=lambda x: abs(x[1]), reverse=True)
        ],
        "pro_coverage": {
            "total": total_proj,
            "mapped": mapped_proj,
            "unmapped": total_proj - mapped_proj,
        },
        "it_projects_linked": len(it_set),
    }
```

### 4.2 模拟计算 `POST /api/linkage/impact-simulate`

```python
PER_PERSON_COST = 10.2  # 部门规定：人均综合成本 8500 元/月 = 10.2 万/年

class SimulateRequest(BaseModel):
    proj_id: int
    hr_change: int  # 正=增加, 负=减少

@router.post("/impact-simulate")
def simulate_impact(data: SimulateRequest, current_user: dict = Depends(require_permission("linkage:read"))):
    db = get_db()
    cur = db.cursor()

    proj = cur.execute(
        "SELECT name FROM professional_projects WHERE id=?", (data.proj_id,)
    ).fetchone()
    proj_name = proj["name"] if proj else "未知项目"
    db.close()

    h = data.hr_change
    return {
        "proj_id": data.proj_id,
        "proj_name": proj_name,
        "hr_change": h,
        "per_person_cost": PER_PERSON_COST,
        "total_impact": round(h * PER_PERSON_COST, 2),
    }
```

## 七、风险与对策略

| 风险 | 影响 | 对策略 |
|------|------|--------|
| 专业项目或 IT 项目被删除后，联动映射出现孤儿引用 | 页面显示异常 | `getMappings` 中 LEFT JOIN 可容错，proj_name 为 null 时显示「(已删除项目)」 |
| 部门人均成本标准调整 | 模拟计算结果需同步更新 | `PER_PERSON_COST` 作为模块级常量，单一修改点，且为固定值不查数据库 |
| 多人同时编辑同一条映射 | 数据被覆盖 | 已采用乐观锁 `version` 字段，参照现有 finance.py 的做法 |
| JSON 字段存储中文乱码 | SQLite 兼容问题 | SQLite TEXT 天然支持 UTF-8，且现项目已有 JSON 处理先例（前端传递、后端 json.dumps 存入） |

- [ ] **数据关联正确**：选专业项目时，下拉列表来自真实数据；选 IT 项目时同理
- [ ] **联动概览准确**：summary 接口返回的聚合数字与表格中的数据一致
- [ ] **模拟计算正确**：增减 N 人的财务影响与人均公式推算一致
- [ ] **异常情况不崩溃**：删除项目后联动映射显示「已删除」，不报错
- [ ] **驾驶舱可见**：第五个卡片展示联动概况
