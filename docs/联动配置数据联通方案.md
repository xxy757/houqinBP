# 四维联动配置 · 数据联通方案

> 宝士制冷 · 后勤部四项规划统筹管理系统

---

## 一、背景

当前系统从 5 个 Excel 文件提取数据，形成四个独立维度：

| 维度 | 标识 | 数据量 | 颜色 |
|------|------|--------|------|
| 专业项目 | `pro` | 17 个项目，6 个处室 | 青色 |
| 信息化方案 | `it` | 18 个子项目，4 大类 | 紫色 |
| 人力资源 | `hr` | 68 人，目标编制 48 人 | 橙色 |
| 财务管控 | `fin` | 22 个科目，总预算 ≈ 2139 万 | 红色 |

**核心问题**：四个维度之间目前**没有任何数据库层面的关联字段**，数据各自由独立 Excel 导入，无法回答跨维度问题（如「增减一个岗位对财务预算的影响」）。

---

## 二、现有硬链接（Excel 天然存在）

### 链路 ①：人力优化 → 财务工资预算

```
优化 11 人 → 工资从 449 万降至 362 万（降幅 24.1%）
```

量化关系（部门规定按统一标准计算）：

```
人均综合成本 = 8500 元/月 = 10.2 万/年
```

**这是目前唯一可自动计算的链路。**

### 链路 ②：人员月度调整计划 → 财务执行时间线

```
逐月逐人的编制变化（M5-M8 集中优化 11 人）
   ↓ 对齐 ↓
财务执行时间线（人员编制与排班方案阶段）
```

目前只是概念对齐，缺少具体的一对一映射。

## 四、核心联动公式

### 公式 1：增减人员对财务的综合影响

```
增减 N 人 → 财务综合影响 = N × 10.2 万/年（部门规定人均 8500 元/月）
```

### 公式 2：IT 系统上线对人力变化的影响

```
Δ人力 = Σ(各 IT 项目人力变化)
      = 新增项目人数(+6) + 替代优化人数(-3)
      = 净增 3 人

Δ财务 = 净增人数 × 10.2 万 + IT 系统一次性投入
      = 3 × 10.2 万 + 一次性采购/实施费
```

### 公式 3：专业项目对财务的净影响

```
项目净影响 = Σ(节支项) - Σ(增支项)

例：光伏项目
  净影响 = 水电费节省(400万) - 折旧增加(15万)
         = +385 万/年（正向节支）
```

## 五、技术实施方案

### 5.1 新增数据库表 `linkage_mappings`

```sql
CREATE TABLE IF NOT EXISTS linkage_mappings (
  id                  INTEGER PRIMARY KEY AUTOINCREMENT,
  proj_id             INTEGER REFERENCES professional_projects(id),
  it_ids              TEXT DEFAULT '[]',
  hr_change_desc      TEXT DEFAULT '',
  hr_headcount        INTEGER DEFAULT 0,
  hr_posts            TEXT DEFAULT '',
  hr_month_start      INTEGER,
  hr_month_end        INTEGER,
  fin_budget_impact   REAL DEFAULT 0,
  fin_subjects        TEXT DEFAULT '[]',
  fin_description     TEXT DEFAULT '',
  responsible_person  TEXT DEFAULT '',
  created_at          TEXT DEFAULT (datetime('now','localtime')),
  updated_at          TEXT DEFAULT (datetime('now','localtime')),
  version             INTEGER DEFAULT 1
);
```

字段说明：

| 字段 | 类型 | 说明 |
|------|------|------|
| `proj_id` | INTEGER | 专业项目外键（核心锚点） |
| `it_ids` | TEXT(JSON) | 关联的 IT 项目 ID 数组，如 `[1,3,8]` |
| `hr_headcount` | INTEGER | 人数变化，正=增加，负=减少 |
| `hr_posts` | TEXT | 涉及岗位名称 |
| `hr_month_start/end` | INTEGER | 变化起止月份（1-12） |
| `fin_budget_impact` | REAL | 预算影响金额（万元） |
| `fin_subjects` | TEXT(JSON) | 影响科目数组，如 `["工资","社保"]` |

### 5.2 新增后端路由 `backend/routes/linkage.py`

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| `GET` | `/api/linkage/mappings` | 获取全部联动配置（含 JOIN 后的项目名） | `linkage:read` |
| `GET` | `/api/linkage/mappings/{id}` | 获取单条配置 | `linkage:read` |
| `POST` | `/api/linkage/mappings` | 创建联动配置 | `linkage:write` |
| `PUT` | `/api/linkage/mappings/{id}` | 更新联动配置 | `linkage:write` |
| `DELETE` | `/api/linkage/mappings/{id}` | 删除联动配置 | `linkage:delete` |
| `GET` | `/api/linkage/summary` | 🔑 联动影响摘要（聚合计算） | `linkage:read` |
| `POST` | `/api/linkage/impact-simulate` | 🔑 模拟计算增减人员对财务影响 | `linkage:read` |

#### `impact-simulate` 接口设计

请求：
```json
{
  "proj_id": 7,
  "hr_change": 2
}
```

响应：
```json
{
  "proj_id": 7,
  "proj_name": "厂区节能降耗项目",
  "hr_change": 2,
  "per_person_cost": 10.2,
  "total_impact": 20.4
}
```

> 说明：部门规定人均综合成本按 8500 元/月（10.2 万/年）统一计算，不再分拆工资/社保/公积金/福利。

```typescript
export interface LinkMapping {

  it_ids: number[]

  total_hr_impact: number
  total_budget_impact: number

    hr_impact: { increase: number; decrease: number; net: number }

  affected_subjects: { subject: string; amount: number }[]

export interface ImpactSimulation {
  proj_id: number
  proj_name: string
  hr_change: number

}
```
